# Phase 5.7 — Astro Product Page Real Content Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the Phase 5.0 placeholder Astro page (`apps/site/src/pages/index.astro`) with a real product page: hero with download CTA, three feature cards, three screenshots/mockups, footer with links. Adds a changelog page and a 404 page. Themed to match the desktop app's neon-on-dark palette via Tailwind.

**Architecture:** `apps/site/` becomes a small Astro static project with Tailwind for styling. A reusable `Base.astro` layout owns `<head>`, the dark theme baseline, and the footer. Three components (`Hero`, `FeatureCard`, `Screenshot`) compose `index.astro`. SVG mockups (not PNG screenshots) under `public/screenshots/` so the page is visually complete on first deploy without needing to capture real app screenshots — the SVGs are deliberately diagrammatic and can be swapped for real PNGs later. `changelog.astro` reads a single Markdown source via Astro's content imports; `404.astro` is a small static fallback.

**Tech Stack additions:** `@astrojs/tailwind` integration, `tailwindcss` (already a transitive dep but pinned here), `@astrojs/mdx` not needed for v1.

**Predecessor:** Phase 5.6 (committed; HEAD `8b9a921`). **Spec:** `docs/superpowers/specs/2026-05-09-phase-5-catalog-config-indexer-design.md` §12.2.

- [ ] **Step 2: Changelog page**

Astro 4 supports importing markdown directly via the `import.meta.glob` pattern OR via `?raw`. Simplest: import the markdown raw and render through a tiny inline parser (no extra deps). For v1, we render headers and paragraphs only — no full markdown features.

Create `apps/site/src/pages/changelog.astro`:

```astro
---
import Base from '../layouts/Base.astro';
import changelogText from '../content/changelog.md?raw';

interface Block {
  kind: 'h2' | 'p' | 'ul';
  text: string;
  items?: string[];
}

function parseBlocks(text: string): Block[] {
  const blocks: Block[] = [];
  const lines = text.split('\n');
  let i = 0;
  while (i < lines.length) {
    const line = lines[i] ?? '';
    if (line.startsWith('## ')) {
      blocks.push({ kind: 'h2', text: line.slice(3).trim() });
      i++;
    } else if (line.startsWith('- ')) {
      const items: string[] = [];
      while (i < lines.length && (lines[i] ?? '').startsWith('- ')) {
        items.push((lines[i] ?? '').slice(2).trim());
        i++;
      }
      blocks.push({ kind: 'ul', text: '', items });
    } else if (line.trim().length > 0) {
      blocks.push({ kind: 'p', text: line.trim() });
      i++;
    } else {
      i++;
    }
  }
  return blocks;
}

const blocks = parseBlocks(changelogText);
---
<Base title="Starlight — Changelog">
  <section class="max-w-3xl mx-auto px-6 py-16">
    <h1 class="text-2xl font-semibold mb-8">Changelog</h1>
    <div class="flex flex-col gap-5">
      {blocks.map((b) => (
        b.kind === 'h2' ? (
          <h2 class="text-[13px] tracking-widest uppercase text-neon-cyan border-b border-line pb-1.5">{b.text}</h2>
        ) : b.kind === 'p' ? (
          <p class="text-[13px] text-muted leading-relaxed">{b.text}</p>
        ) : (
          <ul class="text-[13px] text-muted leading-relaxed list-disc pl-5">
            {b.items?.map((it) => (<li>{it}</li>))}
          </ul>
        )
      ))}
    </div>
  </section>
</Base>
```
