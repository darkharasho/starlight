# Phase 5.2 — Manual Library Entries Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let users add games the auto-scanners missed by picking an executable on disk and giving it a display name. Persisted in `config.manualGames[]` (slot already reserved in Phase 5.1's user-config schema), surfaced in the Library tab via a `ManualScanner` that runs alongside `SteamScanner`.

**Architecture:** A new `ManualScanner` in `library-host.ts` reads `config.manualGames` and yields `DetectedGame`s with `source: 'manual'`. The renderer adds an "Add manually" button on the Library route that opens a native file dialog (via a new `pickExecutable` IPC), prompts for a name, generates a deterministic id (`manual-<slug>-<hash>`), and persists the entry through the existing config store. `library-store` gains `addManual(name, exePath)` and `removeManual(id)` actions that update config + trigger re-scan. Manual tiles get a `×` remove affordance on hover. Boxart for manual entries: best-effort match against the catalog by exe basename; if found, render Steam CDN cover; otherwise neutral placeholder.

**Tech Stack additions:** none — Electron's `dialog`, Zod, Zustand are all already in use.

**Predecessor:** Phase 5.1 (committed; HEAD `e3dcbd0`). **Spec:** `docs/superpowers/specs/2026-05-09-phase-5-catalog-config-indexer-design.md` §7.

```ts
export interface ManualScannerOpts {
  readManualGames?: () => Promise<ManualGame[]>;
  readCatalog?:     () => Promise<CatalogIndex | null>;
}

export class ManualScanner implements LibraryScanner {
  readonly source = 'manual' as const;
  private readonly readManualGames: () => Promise<ManualGame[]>;
  private readonly readCatalog:     () => Promise<CatalogIndex | null>;
  constructor(opts: ManualScannerOpts = {}) {
    this.readManualGames = opts.readManualGames ?? (async () => (await getConfig()).manualGames);
    this.readCatalog     = opts.readCatalog     ?? (async () => getCachedIndex());
  }
  async scan(): Promise<DetectedGame[]> {
    let games: ManualGame[];
    try { games = await this.readManualGames(); }
    catch { return []; }
    if (games.length === 0) return [];
    const catalog = await this.readCatalog().catch(() => null);
    return games.map((g): DetectedGame => {
      const installDir = dirnameOf(g.exePath);
      const match = catalog ? matchCatalog(g.exePath, catalog) : null;
      const out: DetectedGame = {
        source: 'manual',
        appId: g.id,
        name: g.name,
        installDir,
      };
      if (match !== null) out.boxartSteamAppId = match;
      return out;
    });
  }
}

function dirnameOf(p: string): string {
  // Cross-platform-safe basename strip. Works for both POSIX (/) and Windows (\) paths.
  const idx = Math.max(p.lastIndexOf('/'), p.lastIndexOf('\\'));
  return idx === -1 ? '' : p.slice(0, idx);
}

function basenameOf(p: string): string {
  const idx = Math.max(p.lastIndexOf('/'), p.lastIndexOf('\\'));
  return idx === -1 ? p : p.slice(idx + 1);
}

function matchCatalog(exePath: string, catalog: CatalogIndex): number | null {
  const baseLower = basenameOf(exePath).toLowerCase();
  const baseStripped = baseLower.replace(/\.exe$/, '');
  for (const entry of catalog.games) {
    if (entry.steamAppId == null) continue;
    for (const candidate of entry.processName) {
      const c = candidate.toLowerCase();
      const cStripped = c.replace(/\.exe$/, '');
      if (c === baseLower || cStripped === baseStripped) return entry.steamAppId;
    }
  }
  return null;
}
```

```ts
function slugify(name: string): string {
  return name.toLowerCase()
    .replace(/[^a-z0-9]+/g, '-')
    .replace(/^-+|-+$/g, '')
    .slice(0, 32) || 'unnamed';
}

function shortHash(s: string): string {
  // djb2 — deterministic, sync, cross-platform, 6-char base36 output.
  let h = 5381;
  for (let i = 0; i < s.length; i++) h = ((h * 33) ^ s.charCodeAt(i)) >>> 0;
  return h.toString(36).slice(0, 6).padStart(6, '0');
}
```

```ts
addManual: async (name, exePath) => {
  const id = `manual-${slugify(name)}-${shortHash(exePath)}`;
  const cfg = await starlight().getConfig();
  const existing = cfg.manualGames;
  const dedupedRest = existing.filter(g => g.id !== id);
  const next = [
    ...dedupedRest,
    { id, name, exePath, addedAt: new Date().toISOString() },

  await starlight().updateConfig({ patch: { manualGames: next } });
  // Trigger re-scan so the new tile appears.

removeManual: async (id) => {
  const cfg = await starlight().getConfig();

  await starlight().updateConfig({ patch: { manualGames: next } });
  await useLibraryStore.getState().scan();
},
```

```tsx
import { useState } from 'react';
import { starlight } from '../ipc-client.js';

interface Props {
  onCancel: () => void;
  onConfirm: (data: { name: string; exePath: string }) => Promise<void>;
}

export function AddManualGameDialog({ onCancel, onConfirm }: Props): JSX.Element {
  const [exePath, setExePath] = useState<string | null>(null);
  const [name, setName] = useState('');
  const [pickError, setPickError] = useState<string | null>(null);
  const [saving, setSaving] = useState(false);

  async function handlePick(): Promise<void> {
    setPickError(null);
    const r = await starlight().pickExecutable();
    if (!r.ok) {
      if (r.error === 'cancelled') return;
      setPickError(r.message ?? r.error);
      return;
    }
    setExePath(r.path);
    if (!name) {
      const base = r.path.split(/[\\/]/).pop() ?? '';
      setName(base.replace(/\.(exe|app|sh)$/i, ''));
    }
  }

  async function handleSave(): Promise<void> {
    if (!exePath || !name.trim()) return;
    setSaving(true);
    try {
      await onConfirm({ name: name.trim(), exePath });
    } finally {
      setSaving(false);
    }
  }

  return (
    <div className="fixed inset-0 bg-bg/80 flex items-center justify-center z-50">
      <div role="dialog" aria-labelledby="add-manual-title"
           className="w-[420px] rounded-sm border border-line bg-panel px-5 py-4 flex flex-col gap-3">
        <h2 id="add-manual-title" className="text-sm font-semibold">Add a manual game</h2>
        <div className="flex flex-col gap-2">
          <button
            type="button"
            onClick={() => void handlePick()}
            className="self-start px-3 py-1.5 text-xs rounded-sm border border-neon-cyan text-neon-cyan glow-cyan hover:bg-neon-cyan/[0.08]"
          >
            {exePath ? 'Pick a different executable' : 'Pick executable…'}
          </button>
          {exePath && <div className="text-[10px] text-muted truncate">{exePath}</div>}
          {pickError && <div className="text-[10px] text-neon-pink">{pickError}</div>}
        </div>
        <label className="flex flex-col gap-1 text-xs">
          <span className="text-muted text-[10px] tracking-wider uppercase">Display name</span>
          <input
            aria-label="Display name"
            value={name}
            onChange={(e) => setName(e.target.value)}
            disabled={!exePath}
            className="px-2 py-1.5 text-xs rounded-sm bg-bg border border-line text-ink disabled:opacity-50"
          />
        </label>
        <div className="flex justify-end gap-2 pt-1">
          <button
            type="button"
            onClick={onCancel}
            className="px-3 py-1.5 text-xs rounded-sm border border-line text-muted hover:border-neon-pink hover:text-neon-pink"
          >
            Cancel
          </button>
          <button
            type="button"
            onClick={() => void handleSave()}
            disabled={!exePath || !name.trim() || saving}
            className="px-3 py-1.5 text-xs rounded-sm border border-neon-cyan text-neon-cyan glow-cyan hover:bg-neon-cyan/[0.08] disabled:opacity-50"
          >
            Save
          </button>
        </div>
      </div>
    </div>
  );
}
```

**Risks flagged for the implementer:**
- The `dirnameOf`/`basenameOf` helpers in Task 3 are hand-rolled to be cross-platform (POSIX `/` and Windows `\`). Could use `node:path/posix`/`node:path/win32` for more correctness, but for v1 the simple last-separator heuristic is sufficient. If a future user reports a bizarre exe path that breaks parsing, switch to `node:path`.
- `getCachedIndex` accessor is re-introduced in `catalog-host.ts`. If the file's existing dead-code cleanup removed `cachedIndex`, this re-introduction is intentional. The accessor returns `null` until the first successful `fetchCatalog()` — so Library tiles loaded before the catalog finishes fetching may render manual entries without boxart even if a catalog match exists. Acceptable: the next user-triggered Library refresh after catalog load picks up the boxart.
- `window.confirm` for remove is a stop-gap; a proper modal lands later if/when other destructive actions need confirmation.
- `pickExecutable`'s Electron filter (`['exe', 'app', 'sh', '*']`) is permissive on purpose — Linux native binaries have no extension and require the `*` filter. Users can pick any file; there's no validation that the picked file is actually an executable. The downstream behavior (failed engine attach when the user picks `notes.txt`) is the natural error surface.
- Manual entries don't dedupe across name changes — if a user adds `foo.exe` as "Foo" and again as "FooDX", they'll get two tiles since the synthetic id depends on `name`. This is intentional: the spec says dedupe is by `exePath` (via the `shortHash`), but the slug part lets users distinguish two installs of the same exe in different locations. The Phase 4 dedupe-by-exePath test in Task 4 verifies the exact-same-input idempotency, not the cross-name case.
